# سينما برو — Astro Redesign

## نظرة عامة
- **الاسم**: سينما برو
- **الهدف**: إعادة تصميم موقع Astro جذرياً بواجهة عربية RTL حديثة مع الحفاظ على نفس البيانات والمحتوى والروابط الأصلية.
- **الإطار**: Astro Static Site + Tailwind CSS + CSS مخصص.
- **الخط**: Cairo من Google Fonts.
- **الألوان الأساسية**: خلفية `#0B0F19`، سطح `#111827`، أزرق `#2563EB`، بنفسجي `#7C3AED`، نص `#F9FAFB`.

## ما تم إنجازه
- إعادة بناء الهوية البصرية باسم **سينما برو** في الهيدر، الميتا، manifest، وصف الصفحات، وواجهات التفاصيل/المشاهدة.
- استبدال الهيدر القديم بهيدر جديد بالكامل:
  - شريط علوي 40px باللون البنفسجي.
  - هيدر رئيسي sticky بارتفاع 70px وخلفية داكنة مع `backdrop-filter: blur(12px)`.
  - شعار تشغيل `▶` مع اسم الموقع.
  - شريط بحث مركزي بعرض 400px.
  - أيقونات إجراءات فقط في الطرف الآخر.
  - شريط تصنيفات أفقي قابل للتمرير.
- إعادة تصميم الصفحة الرئيسية:
  - Hero بنمط split-screen بنسبة 55/45.
  - مؤثرات CSS particles/waves.
  - عنوان typing-style.
  - أقسام: الأحدث إضافةً، الأكثر مشاهدة، الأنمي، واختر مزاجك.
  - تفعيل فلترة “اختر مزاجك” بدون تغيير البيانات.
- إعادة تصميم صفحات التصنيفات عبر مكون موحد:
  - Hero gradient.
  - Filter pills.
  - Grid متجاوب: 5 أعمدة desktop، 3 tablet، 2 mobile.
- إعادة تصميم صفحات التفاصيل:
  - Hero بخلفية poster/backdrop مع blur وbrightness.
  - Poster مائل.
  - إحصائيات وتقييم progress.
  - Tabs: معلومات، حلقات، مشابهة، تعليقات.
  - Season dropdown بدلاً من tabs القديمة.
- إعادة تصميم صفحات المواسم والحلقات ببطاقات جديدة وروابط مشاهدة مباشرة.
- إعادة كتابة صفحة المشاهدة بنمط Cinema Mode:
  - خلفية سوداء.
  - Player بنسبة 75% على desktop.
  - Sidebar يمين بنسبة 25%.
  - معلومات العمل وروابط السابق/التالي أسفل المشغل.
  - زر “وضع السينما” لإخفاء كل شيء وإبقاء المشغل فقط.
- تفعيل lazy loading للصور حيثما أمكن مع `decoding="async"` و`referrerpolicy="no-referrer"`.
- إضافة Skeleton loading بصري للصور/البطاقات عند غياب الصور.
- تفعيل smooth scroll عام.
- إنشاء favicon SVG وتحديث `site.webmanifest` باسم سينما برو.
- حذف صفحات HTML قديمة داخل `public/movie` لأنها كانت مخرجات بناء قديمة وليست سورس.

## المسارات الوظيفية الحالية
- الصفحة الرئيسية: `/`
- الأفلام: `/movies`
- صفحة فيلم: `/movie/{slug}`
- المسلسلات: `/series`
- صفحة مسلسل: `/series/{slug}`
- صفحة موسم مسلسل: `/series/{slug}/season/{season}`
- الأنمي: `/anime`
- صفحة أنمي: `/anime/{slug}`
- صفحة موسم أنمي: `/anime/{slug}/season/{season}`
- المشاهدة:
  - فيلم: `/watch/{slug}`
  - حلقة: `/watch/{slug}?s={season}&e={episode}`
- البحث: `/search?q={query}`

## بنية البيانات
- مصدر البيانات الأساسي محفوظ كما هو: `src/data/titles/*.json`
- ملفات البيانات المولدة:
  - `src/data/generated/all.json`
  - `src/data/generated/index.json`
  - `src/data/generated/search-index.json`
  - `src/data/generated/stats.json`
- لا توجد قاعدة بيانات؛ الموقع يبنى كملفات ثابتة.
- لم يتم تعديل ملفات البيانات الأصلية حفاظاً على شرط عدم تغيير المحتوى والروابط.

## أوامر التطوير
```bash
npm install
npm run data
npm run dev
```

## البناء الكامل
بناء الموقع الكامل يتم بالأمر:
```bash
npm run build
```

> ملاحظة: بناء Astro الكامل يولّد كل الصفحات الثابتة. بناءً على طلب المستخدم، لم يتم تشغيل البناء الكامل بعد آخر تعديلات الاسم والتصميم؛ يمكن تشغيله لاحقاً عند الانتهاء من جميع التعديلات.

## النشر
- المنصة المستهدفة: Cloudflare Pages.
- مجلد النشر بعد البناء: `dist/`.
- الحالة: لم يتم تنفيذ نشر إنتاجي من هذه الجلسة.

## غير منفذ حالياً
- لم يتم ربط مشروع Cloudflare Pages أو نطاق مخصص.
- لم يتم رفع المشروع إلى GitHub خارجي.
- لم يتم تشغيل بناء كامل بعد آخر تعديلات العلامة التجارية.

## خطوات مقترحة لاحقاً
1. تشغيل تجربة محلية عبر `npm run dev` لاختبار صفحات محددة بدون بناء كامل.
2. بعد اعتماد الشكل النهائي، تشغيل `npm run build` مرة واحدة.
3. اختبار عينات من الصفحة الرئيسية، التصنيفات، التفاصيل، المواسم، والمشاهدة.
4. نشر مجلد `dist/` على Cloudflare Pages.
